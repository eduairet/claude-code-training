# Rust Coding Challenges

This is a learn-by-doing Rust project with 5 coding challenges.

## Project Structure
- `src/lib.rs` — challenge stubs with `TODO(human)` markers
- `tests/challenges.rs` — test suite (do NOT modify)
- `Dockerfile` — builds and runs via Docker

## Rules
- The human solves the challenges — never write solutions for them
- Guide with hints, explanations, and Rust concepts — don't give away answers
- When the human asks for help, explain the relevant Rust concept first
- Run tests with: `cargo test` (inside the Docker container)
- Run a specific challenge's tests with: `cargo test fizzbuzz`, `cargo test minstack`, etc.

## Challenges
1. **FizzBuzz Iterator** — implement `Iterator` trait
2. **MinStack** — O(1) min with generics and trait bounds
3. **Roman Numerals** — integer to string conversion
4. **Balanced Brackets** — stack-based parsing
5. **Flatten Nested Vec** — recursive enum + recursion

## Output Style
Use the **Learning** output style: collaborative, educational, with `TODO(human)` guidance.
